        OPEN-R SDK 1.1.5 r2 からの変更点

OPENR::Fatal() 実行後の EEPROM 保護
  - EEPROM 保護のため、OPENR::Fatal() 実行後は EEPROM に書き込みを行わない。

        OPEN-R SDK 1.1.5 r1 からの変更点

ERS-7 の過電流検出基準の変更
  - 過電流検出とは AIBO がバッテリーからの電流を監視し、過電流が流れた場合に本体を保護するため、
    自分でシャットダウンする（電源を切る）機能です。
    OPEN-R SDK 1.1.5 r1 では、過電流検出の基準をより安全側にしているため、負荷のかかる状態では
    過電流検出してシャットダウンする頻度が高くなります。負荷がかかる状態とは、障害物に AIBO が
    引っかかっている状態や歩きにくい床（毛足の長いカーペットの上など）で使用している状態、無理
    な姿勢を長時間している状態などのことです。
    OPEN-R SDK 1.1.5 r2 の過電流検出の基準は「本体に影響がない」レベルに調整され、過電流検出で
    シャットダウンする頻度も低くなります 


バッテリープランジャーのロック
  - EEPROM 保護のため起動時、終了時にバッテリープランジャーをロックします。
